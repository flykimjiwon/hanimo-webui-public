# Security Policy

Hanimo WebUI is self-hosted software. Please do not publish an unpatched
credential, token, SSRF, upload, or authentication vulnerability in an issue.

Use a private GitHub Security Advisory for this repository when available. If
the repository does not expose that workflow, email Kim Jiwon (김지원) at
`flykimjiwun@naver.com` with the subject `[HANIMO SECURITY]` and include a minimal
reproduction, affected version/commit, impact, and a suggested mitigation.

Do not include real API keys, passwords, private endpoints, user data, or
internal company information in a report.

Supported release claims are limited to the stable Core documented in the
README. Workflow, Screen, RAG, MCP, community, and destructive admin surfaces
are experimental and may have different operational requirements.

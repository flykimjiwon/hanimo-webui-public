# Support policy / 지원 정책

## 한국어

- 설치·사용 질문: GitHub Discussions
- 재현 가능한 버그: GitHub Issues
- 보안 취약점: 공개 이슈가 아닌 GitHub Private Vulnerability Reporting 또는
  `flykimjiwun@naver.com`
- 저작권·귀속·상표 문의: `flykimjiwun@naver.com`

보고할 때 OS, 아키텍처, Docker/Node 버전, Hanimo commit 또는 tag, 재현 단계,
기대 결과와 실제 결과를 포함하세요. 실제 API 키, 비밀번호, 내부 URL, 사용자
데이터는 보내지 마세요.

README에서 Stable Core로 표시된 기능이 우선 지원 범위입니다. Labs 기능은
실험적이며 호환성이나 마이그레이션 보장이 다를 수 있습니다. 응답 시간이나
수정 기한은 별도 유료 지원 계약이 없는 한 보장되지 않습니다.

## English

- Installation and usage questions: GitHub Discussions
- Reproducible bugs: GitHub Issues
- Security vulnerabilities: GitHub Private Vulnerability Reporting or
  `flykimjiwun@naver.com`, never a public issue
- Copyright, attribution, and trademark questions: `flykimjiwun@naver.com`

Include OS, architecture, Docker/Node version, Hanimo commit or tag,
reproduction steps, expected behavior, and actual behavior. Never include real
API keys, passwords, private endpoints, or user data.

The Stable Core identified in the README is the primary support scope. Labs
features are experimental and may have different compatibility or migration
expectations. Response or resolution times are not guaranteed without a
separate paid support agreement.
